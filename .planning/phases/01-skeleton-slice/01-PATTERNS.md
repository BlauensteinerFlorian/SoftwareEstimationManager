## Pattern Assignments — Backend

### `backend/Dockerfile` (infra, multi-stage)

**Analog:** Greenfield — Astral uv official Docker integration pattern.
**Canonical reference:** `01-RESEARCH.md` §"Backend — Dockerfile" (lines 606–644) + §"Pattern 2: Multi-Stage Docker mit uv" (lines 351–381). Source: https://docs.astral.sh/uv/guides/integration/docker/

**Stage skeleton to mirror:**
```dockerfile
FROM python:3.12-slim-bookworm AS builder
COPY --from=ghcr.io/astral-sh/uv:0.11.14 /uv /uvx /bin/
WORKDIR /app
# Cache-mount: install deps from frozen lock BEFORE copying source
RUN --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \
    --mount=type=bind,source=pyproject.toml,target=pyproject.toml \
    uv sync --frozen --no-install-workspace
COPY pyproject.toml uv.lock /app/
COPY app /app/app
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --locked --no-editable

FROM python:3.12-slim-bookworm
WORKDIR /app
COPY --from=builder /app/.venv /app/.venv
COPY --from=builder /app/app /app/app
ENV PATH="/app/.venv/bin:$PATH" \
    PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**Critical constraints (planner must enforce):**
- Base image is `python:3.12-slim-bookworm` — **NOT alpine** (Pango/Cairo on musl is fragile per CLAUDE.md §Decision 3; not required for Phase 1 but locks the pattern).
- `EXPOSE 8000` only — no `ports:` mapping in compose for backend (INFRA-04).
- Two-step `uv sync`: deps first (cacheable), then app code (frequent change).

from typing import Literal

Complexity = Literal["low", "medium", "high", "very_high"]

### `backend/app/routers/estimates.py` (source, FastAPI router; request-response)

**Analog:** Greenfield — FastAPI body-as-Pydantic pattern.
**Canonical reference:** `01-RESEARCH.md` §"Backend — app/routers/estimates.py" (lines 556–582). Source: https://fastapi.tiangolo.com/tutorial/body/

**Full skeleton:**
```python
from fastapi import APIRouter
from app.schemas.estimate import EstimateRequest, EstimateResponse

router = APIRouter()

# D-03: hardcoded Skeleton-Multiplikatoren — NICHT die Phase-2-Basisgewichte.
_SKELETON_MULTIPLIER: dict[str, float] = {
    "low": 1.0,
    "medium": 1.5,
    "high": 2.5,
    "very_high": 4.0,
}

@router.post("/estimates", response_model=EstimateResponse)
def create_estimate(req: EstimateRequest) -> EstimateResponse:
    """Skeleton round-trip: pt = pages × multiplier[complexity]."""
    multiplier = _SKELETON_MULTIPLIER[req.complexity]
    pt = req.pages * multiplier
    return EstimateResponse(pt=pt)
```

**Critical constraints:**
- Multipliers `{1.0, 1.5, 2.5, 4.0}` are **explicitly NOT** the Phase-2 base weights (D-03). Throwaway logic — Phase 2 replaces with `weights.json` + `engine.calculate()`.
- No Decimal handling — Phase-1 float arithmetic only.

### `frontend/nginx.conf` (infra, nginx server block)

**Analog:** Greenfield — ARCHITECTURE.md Pattern 1 verbatim.
**Canonical reference:** `01-RESEARCH.md` §"Frontend — nginx.conf" (lines 922–950) + `.planning/research/ARCHITECTURE.md` §"Pattern 1: Reverse-Proxy via Single Frontend Container".

**Full skeleton:**
```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # SPA: alle Client-Side-Routes fallen auf index.html zurück
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API reverse proxy (Same-Origin aus Browser-POV)
    location /api/ {
        proxy_pass http://backend:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 1m;
        proxy_read_timeout 30s;
    }
}
```

**Critical constraints:**
- `proxy_pass http://backend:8000/api/;` — `backend` is the compose service name (DNS resolution within compose network).
- Trailing slash on both `location /api/` AND `proxy_pass http://backend:8000/api/` — preserves path prefix correctly.
- `try_files $uri $uri/ /index.html` — SPA-fallback for Phase-2 client routes (Pitfall #6).
- `client_max_body_size 1m` — sufficient for Phase 1 JSON; PDFs in Phase 4 are *downloads* (response), not uploads.

- `noEmit: true` — Vite handles emit; `tsc -b` only type-checks.

}

export async function postEstimate(req: EstimateRequest): Promise<EstimateResponse> {
  const res = await fetch('/api/estimates', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(req),
  })

## Shared Patterns

### Same-Origin Reverse-Proxy (No CORS)

**Source:** `frontend/nginx.conf` `location /api/` block + `backend/app/main.py` (NO `CORSMiddleware`)
**Apply to:** All current and future API endpoints
**Rationale:** Browser sees one origin (`http://localhost:3000`); nginx proxies `/api/*` to compose-internal `backend:8000`. Pitfall #16 prevention by topology, not configuration.

### Relative API URLs in Frontend

**Source:** `frontend/src/api/client.ts` — `fetch('/api/estimates', ...)`
**Apply to:** All frontend HTTP calls (current + Phase 2+)
**Rationale:** Pitfall #12. Relative URLs resolve correctly in both dev (Vite proxy) and prod (nginx proxy) — single source of truth.

### Layer Layout Anticipates TECH-01 Separation

**Source:** `backend/app/{routers,schemas}/` directory split
**Apply to:** All future backend modules
**Rationale:** Phase 2 adds `engine/`, `services/`. Phase 3 adds `models/`, `db/`. Strict-layer linting (engine must not import FastAPI/SQLAlchemy/schemas) is Phase-2 work but the directory split prepares for it.

### Version Single Source of Truth

**Source:** `backend/app/__init__.py` (`__version__ = "0.1.0"`) + `backend/pyproject.toml` (`version = "0.1.0"`)
**Apply to:** All version references (FastAPI title, `/api/health` payload)
**Rationale:** D-06 healthcheck payload includes `version`. Single constant prevents drift. Phase 2+ may switch to `importlib.metadata.version(...)`.

### Healthcheck-Gated Service Boot

**Source:** `docker-compose.yml` `depends_on: backend: condition: service_healthy`
**Apply to:** Any future service (e.g., Postgres in cloud variant)
**Rationale:** Pitfall #15. Long-form `depends_on` with healthcheck eliminates the "first page-load 404" race.

### Anchoring-Bias Anti-Pattern (No Defaults on Choice Inputs)

**Source:** `frontend/src/components/EstimateForm.tsx` (complexity dropdown placeholder `<option value="" disabled>`) + `backend/app/schemas/estimate.py` (`complexity: Complexity = Field(...)` — no `default=`)
**Apply to:** All future categorical inputs (complexity dropdowns in Phase 2 parameter table)
**Rationale:** Pitfall #17. Establishing the pattern in throwaway code locks the convention.

### Persistent Bind-Mount Targets via `.gitkeep`

**Source:** `data/.gitkeep` + `config/.gitkeep`
**Apply to:** All future host-mounted dirs
**Rationale:** Docker bind-mounts require the host directory to exist. `.gitkeep` makes the directory trackable in VCS so first-run `docker compose up` doesn't fail.
