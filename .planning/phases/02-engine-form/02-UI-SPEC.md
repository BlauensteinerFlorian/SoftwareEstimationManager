## Spacing Scale

Tailwind v4 default `--spacing` scale (4-pt rhythm) is used unchanged. Declared tokens:

| Token | Value | Tailwind class | Usage in Phase 2 |
|-------|-------|----------------|------------------|
| xs | 4px | `p-1`, `gap-1` | Icon-to-text gap inside buttons, tooltip-trigger padding |
| sm | 8px | `p-2`, `gap-2` | Compact spacing inside form rows (label↔input vertical gap) |
| md | 16px | `p-4`, `gap-4` | Default Card content padding; gap between form fields |
| lg | 24px | `p-6`, `gap-6`, `space-y-6` | Card outer padding; gap between Sections inside `/new` |
| xl | 32px | `p-8`, `gap-8` | Page outer padding (desktop); gap between Dashboard Cards |
| 2xl | 48px | `mt-12` | Major break above Sticky-Footer spacer (so last form field is not hidden by overlay) |
| 3xl | 64px | n/a in Phase 2 | Reserved for future landing-style layouts |

**Exceptions:**
- Sticky-Footer height: `64px` (4× tap-target sized) — fixed shadow band above viewport bottom, satisfies WCAG 2.5.5 touch-target floor on mobile.
- Page max-width on `/new` and `/result`: `max-w-4xl` (896px) centered with `mx-auto`. Wider screens get whitespace, not stretched forms (research/STACK "form-heavy admin/dashboard").

## Typography

Four sizes, two weights. Tailwind v4 utility classes only — no custom font-size config.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Body | 16px | `text-base` | 400 (`font-normal`) | 1.5 (`leading-normal`) | Form input text, paragraph copy, scope text body |
| Label | 14px | `text-sm` | 600 (`font-semibold`) | 1.4 (`leading-tight`) | All form labels, table headers, faktoren list entries, char counters, validation messages, tooltip body |
| Heading | 20px | `text-xl` | 600 (`font-semibold`) | 1.3 (`leading-snug`) | Card titles ("Stammdaten", "Parameter", "PERT-Detail" etc.) |
| Display | 32px | `text-3xl` | 600 (`font-semibold`) | 1.2 (`leading-tight`) | Dashboard Kennzahlen-Card numbers ("102,3 PT" / "15.480,75 €") |

**De-emphasized 14px text:** char counters (`523 / 600`), validation messages, and tooltip body all render at `text-sm` with `text-slate-500 font-normal` for visual de-emphasis. The lighter color + lighter weight differentiates them from the 14px Label without introducing additional font sizes or weights into the contract.

**Locale-formatted numbers** always render in `tabular-nums` Tailwind class (`font-variant-numeric: tabular-nums`) so columns of PT/€ values align vertically in the Phasen-Tabelle.

## Color

Built on shadcn `slate` baseColor (CSS variables, light mode). 60/30/10 split:

| Role | CSS variable | Approx hex (light) | Usage in Phase 2 |
|------|--------------|--------------------|-----------------|
| Dominant (60%) | `--background` | `#ffffff` (white) | Page background on `/new` and `/result`; form input background |
| Secondary (30%) | `--muted` / `--card` | `#f8fafc` (slate-50) for muted surfaces; `#ffffff` for card surface with `--border` `#e2e8f0` (slate-200) | Card containers, Sticky-Footer band, dashboard table row banding |
| Accent (10%) | `--primary` | `#0f172a` (slate-900) | **Reserved-for list below** |
| Destructive | `--destructive` | `#dc2626` (red-600) | Char-counter at ≥95% threshold; form field error border + helper text |
| Warning | custom `--warning` token (`#f59e0b` amber-500 border, `#fffbeb` amber-50 bg, `#92400e` amber-800 text) | added to `index.css` `@theme` block | **Risk banner only** (D-04); per-faktor multiplier badge when `> 1.15` in Dashboard Faktoren-Card |
| Success / positive | not used in Phase 2 | — | Reserved for v2 (e.g. "Schätzung gespeichert" toast in Phase 3) |

**Accent reserved for** (explicit list — never "all interactive elements"):
1. Primary CTA `Button` "Berechnen" on Sticky-Footer (`/new`)
2. Secondary CTA `Button` "Zurück zur Eingabe" on Dashboard (`/result`)
3. Currently-focused form field's focus ring (`ring-2 ring-primary`)
4. Active Select option indicator (Radix native, slate-900)
5. Link-style underline on Tooltip trigger icons (info `?`)

**Forbidden accent usage:** card borders, default-state buttons, card headings, table headers, info icons in resting state. These all use slate-200 (border) or slate-700/900 (text) without the primary token.

**Contrast verification (WCAG AA, all ≥ 4.5:1 for normal text, ≥ 3:1 for large text and UI components):**
- Body text `slate-900` on `white`: 16.1:1 ✓
- Label text `slate-700` on `white`: 11.2:1 ✓
- De-emphasized 14px `slate-500` on `white`: 4.85:1 ✓ (above AA-normal)
- Primary button white text on `slate-900`: 16.1:1 ✓
- Warning text `amber-800` on `amber-50` banner bg: 7.4:1 ✓
- Destructive `red-600` on `white`: 5.9:1 ✓


Exact format per row: `{Faktor-Anzeigename}: {Stufen-Label} (×{Multiplikator mit Komma})`

Examples:
- `Technologie-Reife: günstig (×0,95)`
- `Team-Erfahrung: neutral (×1,00)`
- `Quality/Compliance: ungünstig (×1,15)`
- `Dokumentationspflicht: sehr ungünstig (×1,25)`
- `Projekttyp: Migration (×1,10)`
- `Concurrent Users: 201–1000 (×1,05)`

### Char counter format (INPUT-02 + TEXT-01/02 + specifics)

Format: `{current} / {max}` right-aligned beneath the textarea, rendered at `text-sm` with the de-emphasis color rules below.

Color rules:
- `current < max × 0.95` → `text-slate-500`
- `current >= max × 0.95 && current <= max` → `text-amber-600` (warning preview)
- `current > max` → `text-red-600` + form field shows `Pflichtfeld`-style error border + Berechnen disabled

### German format examples (DASH-09, locked)

| Field | Source value | Rendered string |
|-------|-------------|-----------------|
| PT count | `Decimal("102.3")` | `102,3 PT` |
| € total | `Decimal("15480.75")` | `15.480,75 €` |

Format helpers go into `frontend/src/lib/format.ts`:

```ts
const ptFormatter = new Intl.NumberFormat('de-DE', { minimumFractionDigits: 1, maximumFractionDigits: 1 })

const dateFormatter = new Intl.DateTimeFormat('de-DE', { day: '2-digit', month: '2-digit', year: 'numeric' })
```

Backend serializes Decimal as **already-quantized string** (D-08); frontend MUST `Number(str)` only inside the Intl formatter call and MUST NOT store the parsed Number anywhere (TECH-02 + CALC-06).

### `/new` — EstimateForm page

**Page shell:**
```
<div class="min-h-screen bg-background">
  <header class="border-b border-border bg-card">
    <div class="max-w-4xl mx-auto px-4 py-6">
      <h1>Software-Aufwandsschätzung</h1>
      <p>Vollständige PT-Schätzung …</p>
    </div>
  </header>
  <main class="max-w-4xl mx-auto px-4 py-6 pb-32 space-y-6">  {/* pb-32 = 128px so Sticky-Footer never hides last Card */}
    <Card>…Stammdaten…</Card>
    <Card>…Parameter…</Card>
    <Card>…Korrekturfaktoren…</Card>
    <Card>…Annahmen & Ausschlüsse…</Card>

    <div class="max-w-4xl mx-auto px-4 h-full flex items-center justify-between">

      <Button disabled={!formState.isValid}>Berechnen</Button>
    </div>
  </footer>
</div>
```

**Card 1 — Stammdaten field grid:**
- Two-column responsive grid (`grid grid-cols-1 md:grid-cols-2 gap-4`)
- Field order (top-to-bottom, left-to-right): Projektname (full width), Projekttyp (left) + Kundenname (right), Projekt-ID (left) + Gültigkeit-bis (right), Erstellt-von (left) + Tagessatz (right), Projektskizze (full width with counter)
- Mobile (<768px): single column, same vertical order
- Form-field labels render with `text-sm font-semibold` per the Typography contract

**Card 2 — Parameter field grid:**
- Each of the 5 row-parameters (Pages, Use Cases, Business Objects, Interfaces, Batches) renders as: Label (full row, `text-sm font-semibold`) → side-by-side `<Input type="number" min="0">` (anzahl, ~40% width) + `<Select>` (Komplexität, ~60% width) → tiny "ℹ" tooltip-trigger icon next to the Komplexität label
- Below the 5 rows: 3 single-column fields (Languages number-input 1–10, Roles number-input 1–30, Concurrent Users select 5 stages)

**Card 3 — Korrekturfaktoren grid:**
- 2×2 grid on desktop (`grid grid-cols-1 md:grid-cols-2 gap-4`), single column on mobile
- All 4 Selects: NO `defaultValue`, NO selected placeholder option; placeholder text `— Bitte wählen —`. `aria-required="true"`.
- Form-field labels render with `text-sm font-semibold`

**Card 4 — Annahmen & Ausschlüsse:**
- Two stacked `<Textarea rows={5}>` blocks, each with char counter beneath (counter uses `text-sm text-slate-500 font-normal`)
- Labels: `Annahmen` and `Ausschlüsse` (rendered at `text-sm font-semibold`). Helper text: `Optional. Fließt in den Scope-Text und PDF-Export ein.` (rendered at `text-sm text-slate-500 font-normal`)

**Sticky footer behavior:**
- Always pinned to viewport bottom (`position: fixed`)
- `z-50` (above main content); main content has `pb-32` so the last card scrolls clear
- Button width: `min-w-32` (128px) so the label has breathing room
- On Enter inside any text input: trigger form submit (HTML form default), Sticky button reflects pressed state via `:active`
- After successful `POST /api/estimates`: `react-router` `navigate('/result', { state: { result, inputs } })` — no page reload (D-05)
- On submission: `setItem('estimation:last_creator', form.created_by)` (INPUT-07 / specifics)
- On form mount: `getItem('estimation:last_creator')` and pre-fill `created_by` field if non-empty

## Accessibility Floor (WCAG 2.1 AA)

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | Verified above in Color §; all ratios ≥ 4.5:1 for normal text, ≥ 3:1 for large text and UI |
| Keyboard navigation | Tab order matches visual order (top-to-bottom, left-to-right per Card); Sticky button reachable via Tab from last form field |
| Focus indication | Visible `ring-2 ring-primary` on all interactive elements (shadcn default) — `:focus-visible` only, not `:focus` (no ring on mouse-click) |
| Screen reader labels | Every input paired with shadcn `<Label htmlFor="…">`; required fields carry `aria-required="true"`; tooltips paired via `aria-describedby` |
| Error announcement | Form-field errors use `aria-invalid="true"` + helper text `id` referenced by `aria-describedby` |
| Risk banner | `role="alert" aria-live="polite"` so SR announces it on `/result` mount |
| Numeric inputs | `<input type="number" inputMode="numeric" min="…" max="…">` — pulls native numeric keyboard on mobile |
| Date input | `<input type="date">` native (Tailwind-styled); locale-aware on de-DE browsers |
